## Round 1 Regression Check

### 1. Silent celebration moments
**Status: Still open.**
The `CelebrationToast` component is implemented beautifully (spring-fade entry, glow icon well, auto-dismiss, haptic success fired on mount). It fires perfectly on Day 1's first prediction lock-in. But it does not fire on subsequent predictions, settled matches, tier-ups, streak increments, or rank climbs. The wiring is incomplete. **Reason:** `celebrate()` is called nowhere in the scoring/settle flow, only in the initial onboarding or first-prediction path (if at all). Rounds 1 and 2 both need this fixed.

### 2. No welcome-back summary
**Status: Closed.**
`WelcomeBackBanner.tsx` works perfectly. Appears once per session when returning after >6 hours with ≥1 settled prediction since last visit. Shows "+X points while you were away" or "X predictions settled" with a time summary. Has a dismiss button. Slides in with FadeInDown spring. This is exactly what Round 1 asked for, and it shipped correctly in Round 2.

### 3. No scoring transparency
**Status: Closed.**
`app/scoring.tsx` is a full Emerald Minimalism screen: white cards, emerald accents on icons, six clear rules (exact, correct, miss, boost, streak, weekly), and a worked example. Reachable from Settings → "How scoring works". The page is high-quality. **Minor friction:** Not discoverable from Home or first prediction. A new user might not know it exists. But the feature is there and polished.

### 4. Weak current-user highlight in group standings
**Status: Closed.**
`MemberStandingRow` in `app/group/[id].tsx` applies `isCurrentUser && styles.rowHighlight` which includes `standingStyles.youRail` (a full-height emerald bar on the left) and `standingStyles.youPillText` ("YOU" text). This is unmistakable. The current user's row is instantly findable, even at a glance, even without reading usernames. This shipped correctly in Round 2.

### 5. Pending vs correct visual confusion
**Status: Closed.**
`PredictionPill` in `components/MatchCard.tsx` renders four outcomes:
- **pending:** `surface[2]` (neutral gray) + `lock-closed` icon + "Locked 2–1"
- **exact:** `rgba(0, 166, 81, 0.10)` (soft emerald) + `star` icon + "Exact +10"
- **result:** `rgba(0, 166, 81, 0.10)` (soft emerald) + `checkmark-circle` icon + "Correct +5"
- **miss:** `surface[2]` (neutral gray) + `close-circle` icon + "No points"

These are visually distinct. No confusion. Pending (gray + lock) vs correct (emerald + star) are instantly differentiable. This shipped correctly in Round 2.

## New Findings

- **CelebrationToast provider is mounted but dormant:** The component is built, the context is provided, but `celebrate()` is never called from the prediction lock-in or settle flows. It should fire on every meaningful moment (lock-in, tier-up, streak milestone, rank climb, perfect week), not just Day 1's first prediction. This is the #1 gap between the design intent and the shipping state.

- **Pending pills are *too* muted for a streak game:** The lock-closed icon + neutral gray is clear (not confusing), but it could be more celebratory. A soft emerald tint (like the correct/exact pills) might reinforce that "locked in" is a positive action, not a neutral one. Current players don't know if they should feel good about locking in.

- **Group leaderboard "YOU" row is perfect; main leaderboard still diffuses current-user identity:** In group standings, "YOU" is crystal clear (left rail + pill). In the main leaderboard, there's no such highlighting. I can find myself by scanning names, but it takes a second. Consider applying the same left-rail + pill treatment to the main leaderboard's current-user row for consistency.

- **Welcome-back banner works but doesn't tease the details:** The banner shows raw "+5 points" but not *which* predictions or *how* they settled (exact vs correct). A tease like "+5 points: 1 correct ✓" would draw me into the details.

- **Scoring guide is perfect but buried:** The `app/scoring.tsx` page is so good (clear rules, worked example, proper typography). But it's only discoverable from Settings. A new user's first prediction screen should link to it, or the onboarding should mention it.

- **Tier-up toast is missing:** A new user reaches their first tier-up around Day 15–16 and gets nothing. This is a major morale moment that should fire a `celebrate({ variant: 'tier' })` call with fanfare. Right now it's silent.

- **Streaks reach milestones silently:** Day 7 (one week), Day 14 (two weeks), Day 30 (one month) should all be celebrated. Currently invisible.

- **No "rank climbed" celebration on leaderboard:** When a user climbs ranks, there's a subtle green highlight, but no toast. A "You climbed to #7! 📈" toast would reinforce the gamification.
